# 1. 요구사항 분석 (Requirements Analysis)

## 1.1 프로젝트 개요
- 프로젝트명: Tiny Second-hand Shopping Platform
- 목적: 중고거래가 가능한 웹 플랫폼을 안전하게(Secure Coding) 설계·구현
- 대상 사용자: 일반 회원(구매자/판매자 겸용), 관리자(운영자)

## 1.2 기능 요구사항 (제시된 최소 요구사항 기반)

| ID | 요구사항 | 상세 기능 |
|----|----------|-----------|
| FR-01 | 회원가입/인증 | 회원가입, 로그인/로그아웃, 세션 관리 |
| FR-02 | 상품 등록/조회 | 상품 등록(이미지 포함), 상세조회, 목록조회, 수정/삭제(본인만) |
| FR-03 | 사용자 간 소통 | 상품 기준 1:1 채팅(문의/거래 대화) |
| FR-04 | 악성 유저/상품 차단 | 신고 기능(유저/상품), 관리자 검토 후 차단(정지) 처리 |
| FR-05 | 유저 간 송금 | 포인트(잔액) 기반 송금, 거래 내역 조회 |
| FR-06 | 상품 검색 | 제목/카테고리/가격 범위 기반 검색 |
| FR-07 | 관리자 기능 | 회원 관리(정지/해제), 상품 관리(숨김/삭제), 신고 처리, 거래 로그 조회, 감사 로그 |

## 1.3 사용자 스토리 (일부)
- 사용자로서, 회원가입 후 로그인하여 내 상품을 등록하고 싶다.
- 구매자로서, 판매자에게 채팅으로 문의하고 안전하게 포인트를 송금하고 싶다.
- 사용자로서, 악성 판매자/허위 매물을 신고하고 싶다.
- 관리자로서, 신고된 사용자/상품을 검토하여 정지·삭제 처리하고 싶다.
- 관리자로서, 전체 회원/상품/거래를 모니터링하고 싶다.

## 1.4 비기능 요구사항 (보안 중심)
| ID | 항목 | 내용 |
|----|------|------|
| NFR-01 | 인증/인가 | 비밀번호 해시 저장, 세션 하이재킹 방지, 모든 API에 권한 검증(IDOR 방지) |
| NFR-02 | 입력 검증 | 서버 측 검증(Pydantic), SQL Injection/XSS 방지 |
| NFR-03 | 파일 업로드 안전성 | 확장자/시그니처 검증, 저장 파일명 랜덤화, 실행권한 제거 |
| NFR-04 | 트랜잭션 무결성 | 송금 시 원자적 처리(동시성/이중 지출 방지), 잔액 음수 방지 |
| NFR-05 | CSRF 방지 | 상태 변경 요청에 CSRF 토큰 검증 |
| NFR-06 | 로깅/감사 | 관리자 행위 및 주요 이벤트 로깅(민감정보 제외) |
| NFR-07 | 에러 처리 | 운영 모드에서 스택트레이스/내부 정보 미노출 |
| NFR-08 | 전송 보안 | 보안 헤더 적용(CSP, X-Frame-Options 등), 쿠키 HttpOnly/SameSite |
| NFR-09 | Rate Limiting | 로그인 등 브루트포스 방지용 시도 제한 |

## 1.5 범위 외 (Out of Scope)
- 실제 PG(결제대행)사 연동 결제 — 포인트는 데모용 가상 잔액으로 대체 (회원가입 시 기본 지급 + 관리자 지급 기능으로 대체)
- 실시간 알림(웹소켓 푸시) — 폴링 기반 채팅으로 단순화 가능(구현 여유 시 WebSocket 적용)
